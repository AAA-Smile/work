1\. 互动质量分析（比值指标）

仅看总数会被播放量主导，比值能反映内容“质量密度”。



指标	计算方式	意义

点赞率	like / play	内容受欢迎程度

收藏率	favorite / play	内容有用/值得回看程度

投币率	coin / play	观众深度认同度（投币是B站特色）

评论率	reply / play	讨论热度

互动总分	(like + coin\*2 + favorite\*2 + reply)/play	加权综合互动质量

可实现分析：



按投币率排序，找出“播放不高但人人投币”的宝藏视频。



交互率与粉丝数的关系：大UP主是否更易获得高互动率？



对比点赞率与收藏率，发现“叫好不叫座”或“收藏远高于点赞”的实用型内容。



MySQL 示例（投币率 Top 10）：



sql

SELECT title, play, coin,

&#x20;      ROUND(coin/play \* 100, 2) AS 投币率

FROM bilibili\_videos

WHERE play > 10000  -- 排除极端低播放量干扰

ORDER BY 投币率 DESC

LIMIT 10;

2\. 作者影响力模型

已有“出现次数”和“粉丝数”，可以组合出更立体的指标。



维度建议：



平均播放量：AVG(play) 衡量内容稳定性。



粉丝利用效率：SUM(play) / fans（仅抓取时的粉丝数），高值说明UP主能超越粉丝基数获得大曝光。



跨指标加权排名：自行定义权重，例如 总分 = 0.3\*播放量 + 0.2\*点赞 + 0.1\*投币 + ... 然后按作者聚合排名。



MySQL 示例（粉丝效率 Top 10 UP主，且视频数≥2）：



sql

SELECT author\_name,

&#x20;      MAX(fans) AS fans,

&#x20;      AVG(play) AS avg\_play,

&#x20;      SUM(play)/MAX(fans) AS 粉丝效率

FROM bilibili\_videos

GROUP BY author\_name

HAVING COUNT(\*) >= 2

ORDER BY 粉丝效率 DESC

LIMIT 10;

4\. 时长与用户行为的非线性关系

你已经做了时长分组统计，可以深入：



不同时长组的平均互动指标：点赞率、收藏率等是否随时长变化？



散点图 + 趋势线：duration 与 play 或 投币率 的关系，可能存在最佳时长区间。



SQL 示例（按已有分组计算平均互动率）：



sql

SELECT

&#x20;   CASE

&#x20;       WHEN duration < 300 THEN '1. <5分钟'

&#x20;       WHEN duration < 600 THEN '2. 5-10分钟'

&#x20;       WHEN duration < 1200 THEN '3. 10-20分钟'

&#x20;       WHEN duration < 1800 THEN '4. 20-30分钟'

&#x20;       ELSE '5. >30分钟'

&#x20;   END AS 时长分组,

&#x20;   COUNT(\*) AS 视频数,

&#x20;   ROUND(AVG(like/play)\*100, 2) AS 平均点赞率,

&#x20;   ROUND(AVG(favorite/play)\*100, 2) AS 平均收藏率,

&#x20;   ROUND(AVG(coin/play)\*100, 2) AS 平均投币率

FROM bilibili\_videos

WHERE play > 1000

GROUP BY 时长分组

ORDER BY 时长分组;

5\. 分布形态与异常值探索

播放量/粉丝数的长尾分布：用直方图或者箱线图识别极端头部视频与底部视频。



Z-score 标准化：找出各项指标（如投币数）显著偏离同类的视频，可能是“异常爆款”或“数据可疑”对象。



Top N 占比分析：例如，前10个视频的总播放量占100个总播放量的百分比，反映热度集中度。



SQL 示例（集中度）：



sql

SELECT

&#x20;   SUM(play) AS 总播放,

&#x20;   SUM(CASE WHEN rn <= 10 THEN play ELSE 0 END) / SUM(play) \* 100 AS 前10占比

FROM (

&#x20;   SELECT play, ROW\_NUMBER() OVER (ORDER BY play DESC) AS rn

&#x20;   FROM bilibili\_videos

) t;

6\. 综合评分模型

将多项指标归一化后加权，构建一个“综合热度指数”，更全面地评价视频。



步骤（Python 更方便）：



使用 Min-Max 或 Z-score 标准化 play, like, favorite, coin, reply。



自定义权重（如播放0.5, 点赞0.2, 收藏0.1, 投币0.1, 评论0.1）。



计算总分，然后排序。



此方法可以平衡纯播放量霸榜问题，让高互动但播放量稍低的视频也能脱颖而出。
